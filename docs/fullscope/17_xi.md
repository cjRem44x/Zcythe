# Graphics & Windowing — `@xi::`

`@xi::` is Zcythe's built-in graphics framework. It provides a declarative window, draw, event, font, image, and GIF system backed by SDL2. No `@import` or `zcy add` is required — the compiler detects `@xi::` usage and links `libSDL2`, `libSDL2_ttf`, and `libSDL2_image` automatically.

```
System requirement: SDL2, SDL2_ttf, SDL2_image
  Fedora:  dnf install SDL2-devel SDL2_ttf-devel SDL2_image-devel
  Ubuntu:  apt install libsdl2-dev libsdl2-ttf-dev libsdl2-image-dev
  Arch:    pacman -S sdl2 sdl2_ttf sdl2_image
```

---

## Minimal Example

```zcy
@main {
    win := @xi::window(800, 450, "Hello")
    win.fps(60)
    win.center()

    while win.loop {
        win.frame {
            close => { win.default }
        }
        win.keys {
            key_press => {
                if win.key.code == win.keyval.ESC { @sys::exit(0) }
            },
            key_type => {}
        }
        win.draw {
            win.text(fnt, "Hello World", 100, 100)
        }
        win.clearbg(win.color.black)
    }
}
```

## Window Creation

### `@xi::window(w, h, title)` — Create Window

Creates a window and returns a handle. All subsequent `win.*` calls use this handle.

```zcy
win := @xi::window(1280, 720, "My App")
```

### `win.fps(n)` — Target Frame Rate

Sets the target frames per second. The loop sleeps to match this rate.

```zcy
win.fps(60)
```

### `win.center()` — Center on Primary Monitor

Centers the window on the primary display after creation.

```zcy
win.center()
```

## Window Management

These may be called before or after the main loop.

| Call | Description |
|------|-------------|
| `win.size(w, h)` | Resize the window |
| `win.minsize(w, h)` | Set minimum resizable size |
| `win.maxsize(w, h)` | Set maximum resizable size |
| `win.resize(bool)` | Enable or disable user resizing |
| `win.pos(x, y)` | Move the window to an absolute screen position |
| `win.monitor(n)` | Move the window to display index `n` (0-based) |
| `win.width()` | Returns current window width as `i32` |
| `win.height()` | Returns current window height as `i32` |

```zcy
win.size(800, 450)
win.minsize(400, 300)
win.maxsize(1920, 1080)
win.resize(true)
win.pos(100, 100)
win.monitor(0)

w := win.width()
h := win.height()
```

## Monitor Queries

Query display information before or during the loop.

| Call | Returns | Description |
|------|---------|-------------|
| `@xi::monitors()` | `i32` | Number of connected displays |
| `@xi::monitor_width(n)` | `i32` | Width of display `n` in pixels |
| `@xi::monitor_height(n)` | `i32` | Height of display `n` in pixels |
| `@xi::pri_monitor(n)` | `i32` | Index of the primary monitor (always `0` on most systems) |

```zcy
count := @xi::monitors()
w0    := @xi::monitor_width(0)
h0    := @xi::monitor_height(0)
pri   := @xi::pri_monitor(0)
@pf("displays: {count}, primary: {w0}x{h0}\n")
```

## Main Loop

```zcy
while win.loop {
    # event blocks
    # draw block
    # clearbg (optional — sets background for next frame)
}
```

`win.loop` evaluates to `true` while the window is open and running. The loop exits when `win.running` becomes false (e.g. via `win.default` on a close event).

---

## Event Blocks

### `win.frame { event => { … }, … }` — Window State Events

Handles window-level events. Each arm fires when that state occurs.

| Arm | When |
|-----|------|
| `close => { … }` | User clicked the close button |
| `min   => { … }` | Window was minimized |
| `max   => { … }` | Window was maximized |

`win.default` is the built-in handler that exits the loop cleanly on close, and does nothing for min/max.

```zcy
win.frame {
    close => { win.default },
    min   => { win.default },
    max   => { win.default }
}
```

You can replace `win.default` with your own logic:

```zcy
win.frame {
    close => {
        # save state before quitting
        @pl("bye")
        @sys::exit(0)
    },
    min => {}
}
```

### `win.keys { event => { … }, … }` — Keyboard Events

| Arm | When | State available |
|-----|------|-----------------|
| `key_press => { … }` | A key was pressed (physical key, repeats) | `win.key.code`, `win.key.char` |
| `key_type  => { … }` | A text character was typed (Unicode-aware) | `win.key.char` |

`win.key.code` is a `SDL_Keycode` integer. Compare against `win.keyval.NAME` constants.
`win.key.char` is a `u8` representing the ASCII character, or `0` if non-printable.

```zcy
win.keys {
    key_press => {
        @pl(win.key.char)
        n := win.key.code
        if n == win.keyval.ESC   { @sys::exit(0) }
        if n == win.keyval.ENTER { @pl("enter!") }
    },
    key_type => {
        @pl(win.key.char)
    }
}
```

### `win.mouse { event => { … } }` — Mouse Events

Mouse event block (stub — reserved for future use):

```zcy
win.mouse {
    click => {}
}
```

## Drawing

### `win.draw { … }` — Drawing Block

Everything inside `win.draw { }` is rendered each frame. Drawing calls outside this block have no effect.

```zcy
win.draw {
    win.text(fnt, "Score: 42", 10, 10)
    win.img(background, 0, 0)
    win.gif(anim, 200, 300)
}
```

### `win.clearbg(color)` — Clear Background

Clears the framebuffer with the given color. Called **outside** `win.draw` — it clears the back buffer after the frame is swapped, setting the base color for the next frame.

```zcy
win.draw {
    win.text(fnt, "text", 0, 0)   # drawn on top of whatever was there
}
win.clearbg(win.color.black)      # wipe the back buffer for next frame
```

If `win.clearbg` is omitted, previous frame contents remain (useful for persistent paint effects). If present but called inside `win.draw`, it still fires but clears before other draw calls in that block.

## Text & Fonts

### `@xi::font(name, style, fg, bg, size)` — Load Font

Loads a system font by name using `fontconfig`. The compiler resolves the actual font file via `fc-match`. Returns a font handle.

| Arg | Type | Description |
|-----|------|-------------|
| `name` | `str` | Font family name (e.g. `"Arial"`, `"DejaVu Sans"`, `"monospace"`) |
| `style` | `str` | Style flags: `"NORMAL"`, `"BOLD"`, `"ITALIC"`, `"BOLD\|ITALIC"` |
| `fg` | color | Foreground (text) color |
| `bg` | color | Background color (`win.color.clear` for transparent) |
| `size` | `i32` | Font size in points |

```zcy
fnt := @xi::font("Arial", "BOLD|ITALIC", win.color.white, win.color.clear, 48)
```

Font lifecycle:

```zcy
fnt := @xi::font("monospace", "NORMAL", win.color.green, win.color.clear, 20)
defer fnt.free()   # free is automatic via defer; or call manually
```

Font metric queries (available but optional):

```zcy
w := fnt.width("Hello")    # pixel width of string with this font
h := fnt.height("Hello")   # pixel height
```

### `win.text(font, str, x, y)` — Draw Text

Renders a string using the loaded font at pixel position `(x, y)`.

```zcy
win.draw {
    win.text(fnt, "Player 1", 20, 20)
    win.text(fnt, score_str, 20, 60)
}
```

## Images

### `@xi::img(path)` — Load Image

Loads a PNG or JPEG from disk. Returns an image handle. Supports `catch` for error handling.

```zcy
img := @xi::img("/path/to/image.png")
img := @xi::img("/path/to/photo.jpg") catch |e| { _ => "" }
```

Image method reference:

| Call | Description |
|------|-------------|
| `img.load(path)` | Reload from a new path at runtime — frees old texture, loads new one |
| `img.scale(w, h)` | Override draw size to `w × h` pixels; `0, 0` reverts to natural size |
| `img.width()` | Returns natural pixel width (`i32`) |
| `img.height()` | Returns natural pixel height (`i32`) |
| `img.free()` | Destroy GPU texture and reset handle |

`img.load` and `img.scale` take effect immediately — the next `win.draw` that calls `win.img(img, …)` uses the updated texture and dimensions.

```zcy
img := @xi::img("/path/to/first.jpg") catch |e| { _ => "" }
defer img.free()

# later, mid-loop or before loop:
img.load("/path/to/second.jpg")   # swap texture in place
img.scale(320, 240)               # draw at 320×240 regardless of natural size
```

Draw the image:

```zcy
win.draw {
    win.img(img, x, y)
}
```

## Animated GIFs

### `@xi::gif(path)` — Load GIF Animation

Loads a GIF from disk. All frames are decoded into GPU textures on load. Returns a GIF handle.

```zcy
gif := @xi::gif("/path/to/anim.gif")
```

GIF method reference:

| Call | Description |
|------|-------------|
| `gif.load(path)` | Reload from a new path at runtime — frees old frames, loads new ones; preserves loop/delay/scale settings |
| `gif.scale(w, h)` | Override draw size to `w × h` pixels; `0, 0` reverts to natural frame size |
| `gif.delay(ms)` | Set inter-frame delay in milliseconds — takes effect on the next frame advance (real-time) |
| `gif.loop(bool)` | Enable or disable looping; `false` holds on the last frame |
| `gif.free()` | Destroy all frame textures and reset handle |

All methods are real-time: calling them mid-loop changes behavior on the very next `win.draw` call.

```zcy
gif := @xi::gif("/path/to/anim.gif")
gif.loop(true)
gif.delay(80)
defer gif.free()

# swap mid-loop:
gif.load("/path/to/other.gif")   # new frames, loop/delay preserved
gif.scale(256, 256)              # draw at fixed 256×256
gif.delay(30)                    # speed up
```

Draw each frame (advances automatically by frame timing):

```zcy
win.draw {
    win.gif(gif, x, y)
}
```

## Complete Example

```zcy
@main {
    win := @xi::window(900, 600, "Demo")
    win.fps(60)
    win.center()
    win.minsize(400, 300)
    win.resize(true)

    fnt_lg := @xi::font("DejaVu Sans", "BOLD",   win.color.white, win.color.clear, 64)
    fnt_sm := @xi::font("monospace",   "NORMAL",  win.color.lime,  win.color.clear, 20)
    defer fnt_lg.free()
    defer fnt_sm.free()

    img := @xi::img("/home/user/bg.png") catch |e| { _ => "" }
    defer img.free()

    gif := @xi::gif("/home/user/spin.gif")
    gif.loop(true)
    gif.delay(0)
    defer gif.free()

    score := 0

    while win.loop {
        win.frame {
            close => { win.default },
            min   => { win.default },
            max   => { win.default }
        }

        win.keys {
            key_press => {
                n := win.key.code
                if n == win.keyval.ESC   { @sys::exit(0) }
                if n == win.keyval.SPACE { score += 1 }
                if n == win.keyval.R     { score = 0 }
            },
            key_type => {}
        }

        win.draw {
            win.img(img, 0, 0)
            win.gif(gif, 400, 200)
            win.text(fnt_lg, "Score", 50, 50)
            win.text(fnt_sm, @str(score), 50, 130)
            win.text(fnt_sm, "SPACE = +1   R = reset   ESC = quit", 50, 560)
        }
        win.clearbg(win.color.darkblue)
    }
}
```

## Passing Handles to Functions

Any `@xi::` handle — window, font, image, or GIF — can be passed to a function either **by reference** (mutations visible to caller) or **by value** (mutations stay local to the copy).

### Type annotation syntax

| Declaration | Call site | Behaviour |
|-------------|-----------|-----------|
| `param: @xi::win` | `foo(win)` | by value — copy |
| `param: &@xi::win` | `foo(&win)` | by reference — caller sees all mutations |
| `param: @xi::fnt` | `foo(fnt)` | by value |
| `param: &@xi::fnt` | `foo(&fnt)` | by reference |
| `param: @xi::img` | `foo(img)` | by value |
| `param: &@xi::img` | `foo(&img)` | by reference |
| `param: @xi::gif` | `foo(gif)` | by value |
| `param: &@xi::gif` | `foo(&gif)` | by reference |

### Full example

```zcy
# ── helpers ──────────────────────────────────────────────────────────

# Draw a labelled image + gif side by side — all handles by ref so the
# function can call win.draw, win.img, win.gif, win.text without copies.
fn draw_pair(win: &@xi::win, fnt: &@xi::fnt,
             img: &@xi::img, gif: &@xi::gif, label: str) {
    win.draw {
        win.img(img, 0, 50)
        win.gif(gif, 470, 50)
        win.text(fnt, label, 10, 10)
    }
    win.clearbg(win.color.black)
}

# Swap both resources in place — img and gif by ref so the new paths
# are visible to the caller after the function returns.
fn swap_resources(img: &@xi::img, gif: &@xi::gif,
                  new_img: str, new_gif: str) {
    img.load(new_img)
    gif.load(new_gif)
    img.scale(0, 0)   # 0,0 resets to natural size
    gif.scale(0, 0)
}

# Read-only helper — img and gif by value.
# Mutations like img.scale() compile but only affect the local copy;
# the caller's originals are unchanged.
fn log_dims(img: @xi::img, gif: @xi::gif) {
    @pf("img: {}x{}   gif frames: {}\n", img.width(), img.height(), 0)
    img.scale(999, 999)   # no effect on caller
}

# ── main ─────────────────────────────────────────────────────────────

@main {
    win := @xi::window(940, 580, "handle passing demo")
    win.fps(60)
    win.center()

    fnt := @xi::font("monospace", "BOLD", win.color.white, win.color.clear, 24)
    defer fnt.free()

    img := @xi::img("/path/to/photo1.jpg") catch |e| { _ => "" }
    defer img.free()

    gif := @xi::gif("/path/to/anim1.gif")
    gif.loop(true)
    gif.delay(80)
    defer gif.free()

    # Stage 1 — draw originals via helper (by ref)
    draw_pair(&win, &fnt, &img, &gif, "Stage 1 — original")
    @sys::waist(3000)

    # Stage 2 — swap resources via helper (img + gif mutated by ref)
    swap_resources(&img, &gif, "/path/to/photo2.jpg", "/path/to/anim2.gif")
    draw_pair(&win, &fnt, &img, &gif, "Stage 2 — swapped")
    @sys::waist(3000)

    # Stage 3 — log dims by value (no side effects on img/gif)
    log_dims(img, gif)       # by value, no &

    # Scale directly by ref on the originals
    img.scale(300, 220)
    gif.scale(300, 220)
    draw_pair(&win, &fnt, &img, &gif, "Stage 3 — scaled 300x220")
    @sys::waist(3000)

    # Live loop
    while win.loop {
        win.frame {
            close => { win.default },
            min   => { win.default },
            max   => { win.default }
        }
        win.keys {
            key_press => {
                if win.key.code == win.keyval.ESC { @sys::exit(0) }
            },
            key_type => {}
        }
        draw_pair(&win, &fnt, &img, &gif, "Live — ESC to quit")
    }
}
```

### Rules at a glance

- **By reference** (`&@xi::type` / call with `&handle`): the function receives a pointer. Any `.load()`, `.scale()`, `.delay()`, etc. mutate the original — the caller sees the change immediately.
- **By value** (`@xi::type` / call with `handle` — no `&`): the function receives a copy of the handle struct. Mutations compile normally but only affect the local copy; the caller's variable is untouched.
- Mix freely: a single function can take some handles by ref and others by value.

## Draw Model

`@xi::` uses a **retained draw model**:

- `win.draw { }` items are rendered fresh every frame.
- `win.clearbg(color)` is called **outside** `win.draw` — it clears the back buffer *after* the frame is presented, so the clean slate is ready for the next frame.
- If `win.clearbg` is omitted, previous frame content persists (useful for paint or trail effects).
- Conditional drawing works naturally: wrap any draw call in `if`/`else` and it appears or disappears each frame.

```zcy
show_hud := true

win.draw {
    win.img(background, 0, 0)
    if show_hud {
        win.text(fnt, "HUD", 10, 10)
    }
}
win.clearbg(win.color.black)
```

## Implementation Notes

- **No import required** — the compiler detects any `@xi::` usage and automatically appends `-lSDL2 -lSDL2_ttf -lSDL2_image -lc` link flags.
- **Font resolution** — `_xiOpenFont` calls `popen("fc-match --format=%{file} \"name:style\" 2>/dev/null")` to resolve the actual font file path via the system fontconfig database. Falls back to the literal path string if `fc-match` is unavailable.
- **SDL2 renderer** — all drawing goes through `SDL_Renderer` (hardware-accelerated by default; falls back to `SDL_RENDERER_SOFTWARE`). No OpenGL immediate mode is used.
- **GIF loading** — `IMG_LoadAnimation` decodes all frames; each is uploaded to a GPU texture on load. Frame advancement is driven by per-frame delay metadata plus any `gif.delay()` override.
- **Transparent fonts** — `TTF_RenderUTF8_Blended` renders text with per-pixel alpha blending. A `bg` color of `win.color.clear` gives transparent backgrounds; any opaque color fills the glyph bounding box.
- **`win.clearbg` outside draw** — calls `SDL_RenderClear` then continues into the next iteration; `SDL_RenderPresent` fires inside `_xiFrameEnd` at the end of each loop body.
- **Resource ownership** — `@xi::img` and `@xi::gif` are **not** auto-freed; the user must call `img.free()` / `gif.free()` (typically via `defer`). `@xi::font` is also user-freed via `fnt.free()`.
